# FCOS算法

fcos全称是fully convolutional one stage object detector，其实名字就很明显了，关键词是全卷积&一阶段



整个这里的流程就比较复杂了，和之前纯粹的卷积神经网络/全连接神经网络很不一样。之前的网络是直接接收图像，然后一层层加工，一个init，一个forward，一个backward就结束了，是非常简单直接的端对端，backward还能被自动微分器代替，其实不需要自己手写。



而fcos，包括后面的faster RCNN，不仅网络变得更加复杂，而且环节也增加了，流程如下：



#### backbone+FPN -> Prediction Network -> Inference



### backbone+FPN

这里的backbone+FPN可以理解成是预训练吧，是进行一个对于原图像的特征处理，与常规的不同的是这里的输出，不是只输出一个图层，而是输出对应感受野大小不一样的多个图层。从而让后面的网络能够识别大小不一的不同图像（高度提炼的图像来识别大的目标，浅层卷积来提取小的目标）那么高层卷积还好说，为了让浅层卷积的输出也能较好反映特征，我们在这里进行一个自上而下的变换，将上面的C4、C5等深层卷积层加到C3层上。当然不能直接加，矩阵形状都不一样。变形方法：1*1卷积核改变channel数；上采样的时候，用双插值的方法进行“面积”的扩大，之后就是简单的相加。

从而，我们得到了一些大小不同的，都能反应原图像特征的，卷积输出结果



### Prediction Network

这里应该说是算法的核心了。首先我觉得还是先进行宏观理解，再进行结构的梳理，最后再分析细节。



##### 宏观理解上

首先是宏观理解这里要干什么：fcos是一个anchor-free的端对端算法，预测方式是从点出发，回归得到“这个点在什么物体内”，和“从这个点出发，我要画一个怎么样的框”，最后得到三个输出：识别类型（N+1层网络类别概率），中心度，距离框的四个距离。



结构上，一进来就分成两个stem，一块是就是用于“定位”的全卷积层，一块就是用于“识别”的全卷积层。这两层在结构上没有任何区别，就都是卷积层，区别在后续的反向传播中的参数更新中会渐渐分化（毕竟一个是用于体现“平移不变性”的，一个是用于体现“精准定位”的）。

走完两个并联的层，他们的输入被接到一个中间的连接层，在这里他们将被“塑形”，这也算是我这次学习中体会到的很重要的一个点，不仅是矩阵本身蕴含的信息比较重要，**矩阵的形状本身也很重要**。像是这里就会将cls_stem层的输出结果改造成N+1个channel的输出，来表示这个网络是什么；中心度对应的是“一个”输出；box则是对应四个输出，而很多时候一些“塑形”的过程，其实也就是我们一厢情愿的期待输出过程。这些就分别由不同数量的1*1卷积核来进行实现。然后很神奇的点就来了，用这些得到的输出计算一个loss函数，这个training就结束了。

之后还有一个后处理inference层，但是就无关网络了，是一些结果呈现上的处理，用的也是“算法”



##### 具体实现细节上

首先是正负样本的分配 ：只要这个点落在了真实的物体框内部，它就是正样本，负责预测这个物体。FCOS 设定了不同特征层只负责特定大小的物体（也就是上面的P3P4P5），通过判断回归距离的最大值 $\max(l^*, t^*, r^*, b^*)$ 是否在当前层的范围内 $[m_{i-1}, m_i]$，把大小物体强行分流到不同的图层去预测，巧妙地缓解了重叠冲突。



接着就是训练的损失函数 (Loss)：

1. **分类 Loss：使用了 Focal Loss。**

   它的公式是：$$FL(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$    因为整张图上大部分点都是背景（负样本），如果不加干预，背景产生的 Loss 会很大。这里面加了 $(1 - p_t)^\gamma$ 这个权重，预测得越准（$p_t$ 越接近 1），权重越小。

2. **回归框 Loss：使用了 IoU Loss / GIoU Loss。**

   传统的做法是算四个坐标距离的均方差，但四个坐标是有关联的。用交并比（IoU）直接算面积重合度，不仅能把四个边绑定在一起考虑，而且具有尺度不变性（大框和小框偏离同样的像素，IoU 惩罚是一样的）。

3. **中心度 Loss：**

   由于 FCOS 算法中，只要点在框内就是正样本，这就导致那些在物体边缘上的点也会去预测一个框。这些边缘点预测的框通常质量很差。为了减少这些“低质量框”，引入了中心度，公式如下：

   $$Centerness = \sqrt{\frac{\min(l^*, r^*)}{\max(l^*, r^*)} \times \frac{\min(t^*, b^*)}{\max(t^*, b^*)}}$$

   这个公式的意思是，点越靠近框的中心，上下左右的距离越接近，计算出来的值越接近 1；越靠近边缘，值越接近 0。网络需要额外学习这个值，并用交叉熵来算 Loss

   

最后是 Inference 后处理。这部分其实是在之前的网络中都没有出现过的，用**分类概率 $\times$ 中心度**作为最终的置信度得分。这样一来，那些虽然认对了类别，但因为处于物体边缘导致中心度极低的框，得分就会被拉低。而因为在testing环节，我们最后要输出的是“一个物体一个框”这样的输出，所以最后再过一遍 NMS（非极大值抑制），预测框就出来了。

# Faster RCNN算法

在宏观理解上，Faster RCNN 和 FCOS 有两个本质的区别：第一，它是 **Two-stage** 的；第二，它是 **Anchor-based** 的。它不通过一个个点去回归，而是提前在图上画满参考框，然后去考虑“真实的框和这些参考框差了多少”。



流程如下：

#### Backbone + RPN -> RoI Pooling -> Proposals

#### Proposals + Backbone -> Prediction Head -> Inference



### Backbone + RPN

**1. Backbone**

这部分和之前差不多，就是扔进 ResNet 等主干网络，把图片进行特征提取。不过这里的结果会被用到两次

**2. RPN**

这是 Faster RCNN 的灵魂所在，它的任务是“粗筛”，在不知道是什么物体的情况下，先把可能包含物体的框找出来。

**Anchor：** 在特征图的**每一个像素点**上，RPN 会预设出 9 个不同大小和长宽比的虚拟框，这就叫 Anchor。而这里的anchor的大小、数量、方向就是超参数了

**RPN 的输出：** RPN 会在特征图上进行 $3 \times 3$ 的卷积操作，然后分出两条线：

- **分类分支：** 输出 $2k$ 个值，判断这 $k$ 个框里“有物体”还是“没物体”，不进行更加具体的分类
- **回归分支：** 输出 $4k$ 个值，预测真实的框相对于 Anchor 的偏移量，对框试图进行一些微调



所以从上面就可以看出来这里的RPN基本上就是一个小的FCOS网络，不过功能实现的更加浅一点。这样单纯看可能会觉得奇怪，觉得有点“杀鸡焉用牛刀”，毕竟这是一个本来可以单独handle整个任务的分类网络，然后到这里你来进行一个粗筛。但是结合发展历程来看的话是合理的，RCNN就不说了，fastRCNN中，大家发现最终最耗时间就是Proposals的生成，那个时候用的是ss算法，很耗时间（但是我没有深究过为什么很耗时间，我只知道结果）。而这里用这个网络来代替，很好的“代替”了这部分功能，运行速度快很多。所以很自然的，在这里的RPN结尾也有一层类似inference的操作，因为这里的输出的那些Proposals是要到时候送去原图上让全连接层来进行预测的，那么很多重复的框没有意义。



### RoI Pooling 

这一层就是我之前有提到的，是一个“连接”作用的层，用于对接：RPN 丢过来的几千个候选框，大小和长宽比是完全不一样的。但是，第二阶段的 RCNN 网络最后需要连接全连接层来进行精准分类，而全连接层要求输入的特征大小必须是固定的。如果直接对原图进行 Resize，可能会导致物体的形变。RoI Pooling 的做法是：无论你这个候选框在特征图上框出来是 $15 \times 20$ 还是 $40 \times 30$，我都强行用类似最大池化的方法，把你均匀地划分为 $7 \times 7$ 个网格，每个网格里取最大值。这样一来，所有千奇百怪的框，最终都被“捏”成了固定大小的 $7 \times 7$ 特征矩阵，可以开开心心地喂给后面的网络了。



### Prediction Head

这里就很简单了，因为所有的框已经确定了，前面也有backbone来进行特征的提取，已经卷过原图了，所以直接在这些框上面来进行全连接层的展平、识别就好。



### 一些具体实现细节

和 FCOS 一样，这里也主要是**正负样本分配**、**loss**以及**inference**

#### 1. Target Assignment

在 FCOS 里，一个点只要掉进真实的框里，它就是正样本。但在 Faster RCNN 里，规则变了，一切都要靠 **Anchor 与 GT 框的 IoU** 来决定。

**RPN 阶段的分配：** 图上密密麻麻画了那么多 Anchor，谁来负责预测 GT 框？

- **Positive：** 如果一个 Anchor 和某个 GT 框的 IoU $> 0.7$（或者它是所有 Anchor 中与该 GT 框 IoU 最大的），那它就被标记为正样本，负责回归这个 GT 框。
- **Negative：** 如果 IoU $< 0.3$，它就是纯纯的背景。
- **忽略样本：** IoU 在 $0.3 \sim 0.7$ 之间的框，属于“模棱两可”的鸡肋，训练时直接扔掉，既计算 Loss 也不更新参数。

#### 2. Loss Function

Faster RCNN 是一个多任务网络，无论是第一阶段的 RPN，还是第二阶段的fast RCNN，它们的 Loss 结构其实是一模一样的，都是 **“分类 Loss + 回归 Loss”**。它的总损失函数可以写成这样一个联合优化的公式：

$$L(\{p_i\}, \{t_i\}) = \frac{1}{N_{cls}} \sum_{i} L_{cls}(p_i, p_i^*) + \lambda \frac{1}{N_{reg}} \sum_{i} p_i^* L_{reg}(t_i, t_i^*)$$

- **$L_{cls}$：** 就是常规的交叉熵损失。在 RPN 阶段是二分类（是不是物体），在 RCNN 阶段是多分类（具体是哪种物体）。$p_i$ 是预测概率，$p_i^*$ 是真实标签（正样本为 1，负样本为 0）。

- **$p_i^{*} L_{reg}$：** 注意这里乘了一个 $p_i^*$，这算是一个细节，只有正样本才需要算回归 Loss

- **$L_{reg}$：** Faster RCNN 这里使用的是 **Smooth L1 Loss**

#### 3. Inference 

1. **第一次在 RPN 之后：** 上面已经写了
2. **第二次在 RCNN 之后：** 经过第二阶段的精细化微调和具体分类后，再对每一类别的框各自进行一次 NMS，得到最终那几个干干净净、准准确确的预测结果。
