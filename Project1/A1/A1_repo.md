## 二、完成KNN分类器的程序

在这里我体验到了真实的最原始的图像识别技术的具体实现。

KNN的整体流程很简单，从training和testing两块来说：

### training:

他其实没有严格意义上的所谓训练，只要有人给所有数据贴上了标签，那么通过

```pytorch
self.x_train = x_train
self.y_train = y_train
```

两句话其实就算是实现了training，在后续计算中，所有的已标记数据直接被调用就行

### testing:

这也是所谓的KNN被诟病的一个点，老师上课也提出，他反而在testing的时候，出现了相关的O(n)的时间复杂度

具体实现方式上也很简单，就是让待测数据和我们已经标记好的数据进行一个“距离”上的计算。这里距离分为L1和L2，但是实质上大差不差，一个是绝对值作差，一个是欧氏距离。后者会让最后的K分类图上边界更加平滑一些。在本次实现中我们用的是欧氏距离

这里先说一些直观上的理解：人们在试图找一种方式，来确定test集中的一个新图像应该是什么。而就是这里所谓的K邻近点，就是让新样本和训练集去对比，然后我们看他和谁最像，那么自然就是which type的了。

而这一整个过程中要确定的就是“像”怎么被“量化”，那么这里就是用L1/L2两种距离来量化。自然这两种距离其实完全没有任何道理，只能说明“某种程度“上的”像“，但是具体是哪种程度上的？无法被解释。而自然这里得到的结果也不会很好。但是这确实是一种尝试。而且应该也就是由此引出了后续的一些新的方法，人们开始更多的来在算法中增加一些”可解释性“的东西。

除了直观理解之外，就是从具体代码中来的一些感悟了：

### code

这一块三个不同的函数，实现相同的功能还是比较能直观的让人感受到并行计算的威力的，时间从最长的5s+能降到0.1s-，那空间换时间，而底层的矩阵运算又被人们通过GPU等硬件&一些算法所优化，那么就快了很多。比较神奇，这一块在我之前的ZZB_DOAs中的工作中也有用到，当时只当是ai给我一个神奇的加速方式，但是没想到居然也是这里的一种算法核心

不过这里的矩阵变换还是有一点数学推导要求的



```
def knn_cross_validate(
    x_train: torch.Tensor,
    y_train: torch.Tensor,
    num_folds: int = 5,
    k_choices: List[int] = [1, 3, 5, 8, 10, 12, 15, 20, 50, 100],
):
```

还有一块比较想说的就是交叉验证机制了。这一块还是比较神奇的。因为老师上课反复提到，test集最终只能跑一次（因为跑完你的数据库理论上就被污染了）（虽然在KNN这里应该没有），交叉验证在这里实现的是一个讲KNN进一步完善的过程：因为我们刚刚上面其实是实现了在”特定K“下，我们的数据最终出来的predict效果如何，那么对于这一整个训练集，怎么样的K的值是最好的呢？

当然方法很简单，只要测试不同的K就行了。比较严谨的是在测试过程中”training“和”testing“集的得到。因为前面说了test你不能随便test，所以这里将这个training集进行了复用，一部分用于training，一部分用于test，从而比较不同K值下的predict效果

folds就是进行切分，num_folds = 5是每个K测五次，分别用不同的training和test（通过folds的值不同来实现）
