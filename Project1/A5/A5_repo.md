# RNN

学到这里我已经看过transformer了，所以对于这个过渡期的RNN实在兴趣不是很大，就简单记录一下



首先是这个在干什么事情：之前我们只进行了图像的识别，而在这里则是接上了一个RNN，可以看作是一个很简单的NLP，然后实现一个“看图说话”的过程。在这个网络中，特别是注意力机制被引进之前，“看图”和“说话”是有强时序区分的，整个网络就是纯粹的先卷积提取，然后再进行一个纯粹的RNN循环，用来输出一整句话。



整个代码逻辑上实现了三块东西：RNN；LSTM；atten_LSTM。



#### RNN

核心逻辑其实很简单，可以理解成是有一个for循环，串行逻辑（这也恰恰是这个算法最大的问题所在，把上限锁死了），每一次下一个词的预测取决于两件事：上一个词（短期记忆）和 之前的所有“效应和”（长期记忆）。这个想法本身就很朴素，然后实现方式也很简单，就是纯粹的相加：$$h_t = \tanh(x_t \cdot W_x + h_{t-1} \cdot W_h + b)$$，tanh是一个归一化挤压，防止加法导致的数量爆炸。简单的递推导致维度消失的很快，并且两个系数矩阵是全局共用的。



#### LSTM

gemini说LSTM相较于原始的RNN有了很大的改进，但是我看来核心想法是不变的，就还是一个长期记忆和短期记忆的共同作用，只不过这里的长期记忆储存的更加复杂了一点（我都不觉得是更加严谨了，就是复杂了一些），引入了四个门控：Input Gate（输入门）Forget Gate（遗忘门）Output Gate（输出门）Candidate Memory（候选记忆），然后通过$$next\_c = f \odot prev\_c + i \odot g$$来进行长期记忆的更新，然后当前的输出：$$next\_h = o \odot \tanh(next\_c)$$还是一样，是用长期记忆和短期记忆进行一个整合、输出。而通过门控对于记忆进行单独控制，让其不受连乘的影响



#### Atten_LSTM

在这里引入了最原始的单头注意力探测，这里其实不能用后面那种更加“合理的”注意力机制来理解，其实更多的是一种全局的“看图说话”。因为前面两个都是“强分离”，之前得到的图像只是在最开始作为一个输出，之后就没有用到了。但是在这里，网络能够一直“看着图片”。

具体实现方式是：$$A_{gate} = x \cdot W_x + prev\_h \cdot W_h + \mathbf{attn \cdot W_{attn}} + b$$

没错，就是只是多了一项。而且这一项其实和最开始多层感知机中的L1、L2距离是一样的，就是能一定程度上进行反应，但是背后深究的话其实不合适，相当于是有了这个想法，但是算法没有能够匹配上。讲解一下这个式子的话就是单纯的将上一次的输出prev_h，去和提取过的网络进行一个点积，点积完了之后的矩阵，哪个部分大，哪个部分小，其实就是一个很好的“视觉焦点”的体现，那么我们将这个“视觉焦点”也进行一个加权求和变换，放进最终的求和中。



#### Teacher Forcing

这可能是一个RNN的special的训练方式，因为刚刚说了是串行的网络，为了提高训练速度，在训练的时候，是“拿着答案“去看的：训练时用于生成下一个字的数据来源是“上一个字的答案”+“长期记忆”。这样能让一开始的网络加速很多，但是我反正觉得怪怪的，到了后期不一定有用，当然这里还是只讲了最最基础的，可以有很多变体，但是也只是优化，不是质变。

### 宏观理解

先要引入一些概念：

词向量（词嵌入 v.）：在这里，我们的词语都被一系列的高维向量进行编码（可能是1k+维度），但是注意这里不是词典，one-hot的词典机制太浪费空间的，而后期可动的空间也很小。这里的某一“维”其实可以理解成是一个“意思”（维度概念在这里就可以理解成是一组正交的向量），比如“性别”这样。而数学上的定理证明，当维度够高的时候，我们如果将“正交”概念放宽，变成是“几乎正交”，也就是两个向量之间的夹角是89度-91度之间的话，那么此时满足条件的向量数，是几乎和exp(维度)成正比的，也就是100维的空间，其实可以几乎得到$k·e^{100}$大小的语义，所以这既是对于词向量的定义的合理性的解释/印证；一定程度上也对于模型的规模越大，效果越好的一种解释。

位置变量：因为在transformer中的自注意力训练过程中是丢失了位置信息的（并行逻辑），所以这里引入了一个位置的维度，用于表示词语的上下文位置，这其实也相当于是一层“语义”了，到时候是和词向量进行融合的



然后就是顺着网络往下走：

第一层：k、Q、V矩阵的获得。这里要讲一讲这三个矩阵的含义，当然都是“词向量经过训练好的权重矩阵之后”得到的K、Q、V矩阵：

可以想象一个这是一个词向量的三个信息，“Q”表示他身上带着的一些“问题”（quary）；“K”表示他面对别人来问问题，会做出怎么样的回答（Key）；而“V”则表示他的value，到时候真正参与计算的是V。这里但看这些定义很抽象，把atten里面发生了什么看完：

第二层：Attention层。在这里的一个公式是：$$Attention(Q, K, V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$$，最最重要的就是这里的一个Q和K的相乘，其他都是“手段”，QK^T是“意义”：而这份意义很简单，就是每个人的问题，和每个人的答案进行一个点乘。

点乘在向量中的意义其实是衡量两个向量的匹配度的（值越大，越匹配），所以我们可以认为是在语句内部进行一个互相的询问，如果有谁的“回答”和某词的“问题”匹配度很高，那么他们对应的那个点的值就会更大，到时候的V的乘积就大，就能够反应所谓的“注意力”，同时这一过程也很简单的就解决了上下文问题：因为是“乘法”，完全的并行操作，不涉及任何时间，非常直接。

然后在这里还有一些要解释的，因为哪怕仅仅只是将宏观理解其实也有不少细节：一个是词的先后顺序，就导致了在前面的词的“Q”，其实根本没有和后面词的K相匹配的机会（一定程度反映了时序，但是其实是用空间来实现的），所以在一整个K*Q的大矩阵里，有一半的矩阵全是“0”（其实是负无穷，softmax之后变成0，但就是不考虑/问不到的意思）；还有一个是这里的“自注意力”的“自”，其实这是相对于transformer原本的用途：翻译，来说的。（这里我觉得引入“翻译”之后对于上文中的“问答”理解起来就好很多），在翻译中，“Q”是译文的，“K”是原文的，那么就很显而易见了，其实是一个很直观的操作，译文在一一匹配原文。而将这一注意力机制的匹配引入一句话内部，“自己翻译自己”，也就是进行了上下文信息的提取。

之后这里的attention网络的产出就是一个“QK^T”相乘之后的归一化“答案表”（softmax/根号下dk，就都只是数学手段归一化，变成概率）去乘以全部的V，然后就得到了比如说，对于第一个词，他在全文中的注意力应该放在哪里的结果表。

e.g.: it is a dog .这句话中，比如（超级理想，仅用于理解），经过这里的atten之后，$V_{it} = 0.15V_{it} + 0.25V_{is} + 0.6V_{dog}$。表明这里的V向量被更新了，得到了这个词在全文中的含义向量。dog的权重比较大，说明这里的it更像是指代动物的，犬类的什么什么的这种意思会多。

第三层：残差网络：这里进行的事情很简单，就是刚刚得到的Vit+原本的Vit。但是这里背后的解释性很高。最最简单的意思就是“上下文的意思和原意相加”，但是“意思的相加”这个理解很违和，因为感觉你在进行“意思的融合”这种nb的操作，但是用的是最最简单的加法？这很奇怪，为什么加法能融合。这里就要要将一个最开始的词向量那里的概念。我觉得这里我们还是要想象成是“向量”：

先举一个例子：刚刚说了，可能有一个维度是代表的“性别”，这不是乱说的，在训练好的网络中，科学家发现，"man" - "women" + "uncle" = "aunt";"man" - "women" + "king" = "king"（都是约等于），这就体现了，在这个高维空间中的一些方向，是编码了一些“意思”的。那么从向量的角度来理解刚刚的加法就很好理解了：这不是一个杂糅，而是原本的词向量，首尾相加，往“上下文想要你去的方向”进行了一个偏移。比如“dog”被"cute"进行偏移的话，这只狗是杜宾犬的可能性就比藏獒的可能性会大一点，在空间中的“dog”向量往“杜宾犬”偏了，离“藏獒”远了。所以这里的简单的相加这样理解就舒服很多

第四层：全连接层：这是我当时最不理解的地方。这里的全连接层是“针对词自己的”，也就是全都是闭门造车。gemini说是自己对于上述得到的语义的深度消化，同样，这个理由不直观，不舒服。后来在3b1b的视频的标题中我找到了很舒服的解答（当然视频里讲的更清楚）“大模型存放记忆的地方”。就是这里的全连接层很简单，是一个线性层+一个ReLU+一个线性层。那么我还是希望能够带上一种“问答”的感觉来理解这里的变化：在第一个线性层中（假设已经训练好了），同样是进行一个对于刚刚送进来的、融合了上下文意思和自身意思的词向量，进行一系列提问。问题是怎么来的？一次次训练来的，所以叫做“模型的记忆”。因为在这里我们的关注主体一直是那个被反复加工的词向量，因为他在一次训练中就发生了多次变化。但是其实这里的这些“npc”也一直在反向传播的时候进行着更新，所以他们会一点点变成“经验老道”的面试官，会准备好一系列问题（一般是词向量长度的四倍），然后对于词向量进行提问，相乘即得结果。之后就是ReLU，没什么好说的，引入非线性，隔开两个线性层（不然就是一个了），也可以当成是对答案质量进行一个“筛选”。再之后就是第二个线性层，也就是大模型的“V”，和刚刚的$$Attention(Q, K, V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$$是一样的其实，得到了QK的这张批完的卷子，还要去找V进行兑换。兑换下来给你什么？存放在大模型的记忆里。

第五层，再进行残差连接。理由同第三层，将大模型的记忆也塞进你的词向量。

然后这样其实就走完了。但是其实上面说的都是理想情况，这一问答循环可能进行很多很多次，但是就是这样，我们的这个当前的词向量就会越来越准确，对上下文的把控就越来越精准，然后就能通过这里最后一个全新的词向量，输出下一个词了。

##### 以上就是我对于自注意力机制的宏观理解了，多头其实只是给矩阵升维然后同时进行不同的注意力训练。当然这里看下来可能会被transformer的机制搞得有点混，但是其实它本身有一个很大的优点就是可以进行并行计算。向上文中，一句话中从来没有什么“先后”的区别，完美适配GPU的并行计算机制，这其实是他能成功的很大一个点。还有一个并行能实现的很重要的原因是前面rnn里提到的teacher force的训练逻辑，训练的时候每次只预测一个词，前面正确答案都给你，所以拿着一个句子样本的训练量也上去了，并行也可行了。





### 细节实现

这里实在是写不动了，让gemini进行了简单的记录：

------

### 一、 数据预处理与词表构建 (Data Handling)

这部分负责将人类可读的字符串转化为模型能看懂的张量输入。

- **`generate_token_dict(vocab)`**:
  - **实现细节**: 使用 `enumerate` 遍历词表列表，构建一个 `{'token': index}` 的哈希字典，将字符串映射为唯一的非负整数。
- **`prepocess_input_sequence(input_str, token_dict, spc_tokens)`**:
  - **实现细节**: 它的巧妙之处在于对字符的切分逻辑。它通过空格分隔单词，对于在 `spc_tokens` (特殊字符，如 'BOS') 中的词整体映射；对于数字字符串（如 '0333'），它会二次遍历内部字符，将其拆解为独立的数字位（'0', '3', '3', '3'）进行映射。这对于算术类任务非常重要。
- **`AddSubDataset` (类)**:
  - **实现细节**: 继承自 `torch.utils.data.Dataset`。在 `__getitem__` 每次取数据时，它不仅会调用上面的函数将字符串转为 token IDs，还会**实时调用**传入的位置编码函数，返回 `(输入序列, 输入位置编码, 目标序列, 目标位置编码)` 的四元组。

### 二、 核心运算：注意力机制 (Attention Mechanism)

这是 Transformer 的心脏，代码从最基础的循环到优化的张量运算均有实现。

- **缩放点积注意力 (Scaled Dot-Product)**:
  - `scaled_dot_product_two_loop_single` & `scaled_dot_product_two_loop_batch`: 这两个是**教学向**的纯 `for` 循环实现，主要为了演示 $Q \cdot K$ 的点积计算和 Softmax 加权过程。
  - **`scaled_dot_product_no_loop_batch`**: 这是**工业级**实现。使用 `torch.bmm` 进行批量矩阵乘法计算得分。关键细节在于**掩码 (Mask) 的处理**：使用 `scores.masked_fill(mask, -1e9)` 将需要遮挡的位置填充为极小值，这样 Softmax 之后这些位置的权重就会逼近于 0。
- **`SelfAttention` (单头注意力 类)**:
  - **实现细节**: 使用三个 `nn.Linear` 分别生成 Q, K, V。**重点**：代码手动实现了类似 Xavier/Glorot 均匀分布的权重初始化策略，基于公式 $c = \sqrt{6 / (D_{in} + D_{out})}$ 来确定均匀分布的边界 $[-c, c]$。
- **`MultiHeadAttention` (多头注意力 类)**:
  - **实现细节**: 这里没有采用常规的“大矩阵切分”做法，而是直接用 `nn.ModuleList` 实例化了多个 `SelfAttention` 模块。前向传播时遍历这些单头模块，将它们的输出通过 `torch.cat(..., dim=-1)` 在特征维度暴力拼接，最后通过一个 `nn.Linear` 映射回原始维度。

### 三、 网络层与基础组件 (Transformer Building Blocks)

- **`LayerNormalization` (类)**:
  - **实现细节**: 这里**没有**直接使用 `nn.LayerNorm` 或 `torch.std`，而是完全手动根据公式计算均值 `mean` 和方差 `var`。定义了两个可学习参数 `self.gamma` (全1初始化，用于缩放) 和 `self.beta` (全0初始化，用于平移)。
- **`FeedForwardBlock` (类)**:
  - **实现细节**: 标准的两层 MLP (`linear1` -> `ReLU` -> `linear2`)。同样采用了自定义的均匀分布权重初始化。
- **`EncoderBlock` (类)**:
  - **实现细节**: 组装了 `MultiHeadAttention`、两层 `LayerNorm`、`FeedForwardBlock` 和 `Dropout`。在 `forward` 中，严格遵循了带残差连接的计算流：$x' = Dropout(Norm(x + MHA(x)))$，然后再经过 FFN 走一遍类似的残差和归一化逻辑。

### 四、 解码器专属组件 (Decoder Specifics)

- **`get_subsequent_mask(seq)`**:
  - **实现细节**: 极其核心的因果掩码生成器。先生成全 1 矩阵，利用 `torch.triu(..., diagonal=1)` 提取上三角部分（主对角线不保留），然后通过 `.bool()` 转为布尔值，并用 `unsqueeze` 和 `expand` 扩展至整个 Batch 的维度 `(N, K, K)`。
- **`DecoderBlock` (类)**:
  - **实现细节**: 包含了双重注意力机制。第一层是带 Mask 的 `attention_self`（防止看到未来信息）；第二层是**交叉注意力 (Cross-Attention)** `attention_cross`，重点是它的 Q 来自 Decoder 第一层的输出，而 K 和 V 则来自于 Encoder 的输出 `enc_inp`。

### 五、 全局架构组装 (Global Architectures)

- **`Encoder` & `Decoder` (类)**:
  - **实现细节**: 使用 `nn.ModuleList` 在 `for` 循环中堆叠指定数量 `num_layers` 的 Block。Decoder 的尾部多了一个 `proj_to_vocab` 的全连接层，用于将高维特征映射到词表大小的空间中准备输出。
- **位置编码 (Positional Encoding)**:
  - `position_encoding_simple`: 简单粗暴的线性递增位置编码 ($n / K$)。
  - `position_encoding_sinusoid`: 论文原版的正弦/余弦编码。**实现亮点**：为了防止分母上出现极大的次幂导致数值不稳定，代码利用了对数特性 `exp(arange * -(log(10000.0) / M))` 来计算频率除数项。通过切片 `pe[:, 0::2]` 赋值正弦，`pe[:, 1::2]` 赋值余弦。
- **`Transformer` (终极总装 类)**:
  - **实现细节**: 负责统筹全局。实例化了 `nn.Embedding` 将 Token ID 转为向量并加上位置编码。**极其关键的一步**在于解码器的输入处理：`a_emb[:, :-1] + ans_pos[:, :-1]`，通过切掉最后一个字符实现向右平移 (Shifted Right)，确保模型在预测第 $t$ 步时，只能看到 $t-1$ 步及之前的目标输入。

### 六、 辅助与优化模块 (Utilities & Losses)

- **`LabelSmoothingLoss`**:
  - **实现细节**: 为了防止模型过度自信导致的过拟合，代码手动实现了标签平滑。它将绝对的 One-Hot 标签（非0即1）按 $\epsilon$ 的比例分配给了其他所有类别，使得真实标签的概率变成了 $1 - \epsilon$，其余类的概率均摊了 $\epsilon$。
- **`CrossEntropyLoss`**: 对 PyTorch 原生 API 的简单封装。



### transformer的testing运行逻辑

因为上文中的是训练逻辑，transformer的并行架构确实完全适合训练，但是预测阶段，输出的时候只能串行了，这和并行架构的transformer似乎是矛盾的。但是实际上就还是一样，一个词一个词往外蹦，只不过就只需要计算最后一个词的注意力融合了。
