# RNN

学到这里我已经看过transformer了，所以对于这个过渡期的RNN实在兴趣不是很大，就简单记录一下



首先是这个在干什么事情：之前我们只进行了图像的识别，而在这里则是接上了一个RNN，可以看作是一个很简单的NLP，然后实现一个“看图说话”的过程。在这个网络中，特别是注意力机制被引进之前，“看图”和“说话”是有强时序区分的，整个网络就是纯粹的先卷积提取，然后再进行一个纯粹的RNN循环，用来输出一整句话。



整个代码逻辑上实现了三块东西：RNN；LSTM；atten_LSTM。



#### RNN

核心逻辑其实很简单，可以理解成是有一个for循环，串行逻辑（这也恰恰是这个算法最大的问题所在，把上限锁死了），每一次下一个词的预测取决于两件事：上一个词（短期记忆）和 之前的所有“效应和”（长期记忆）。这个想法本身就很朴素，然后实现方式也很简单，就是纯粹的相加：$$h_t = \tanh(x_t \cdot W_x + h_{t-1} \cdot W_h + b)$$，tanh是一个归一化挤压，防止加法导致的数量爆炸。简单的递推导致维度消失的很快，并且两个系数矩阵是全局共用的。



#### LSTM

gemini说LSTM相较于原始的RNN有了很大的改进，但是我看来核心想法是不变的，就还是一个长期记忆和短期记忆的共同作用，只不过这里的长期记忆储存的更加复杂了一点（我都不觉得是更加严谨了，就是复杂了一些），引入了四个门控：Input Gate（输入门）Forget Gate（遗忘门）Output Gate（输出门）Candidate Memory（候选记忆），然后通过$$next\_c = f \odot prev\_c + i \odot g$$来进行长期记忆的更新，然后当前的输出：$$next\_h = o \odot \tanh(next\_c)$$还是一样，是用长期记忆和短期记忆进行一个整合、输出。而通过门控对于记忆进行单独控制，让其不受连乘的影响



#### Atten_LSTM

在这里引入了最原始的单头注意力探测，这里其实不能用后面那种更加“合理的”注意力机制来理解，其实更多的是一种全局的“看图说话”。因为前面两个都是“强分离”，之前得到的图像只是在最开始作为一个输出，之后就没有用到了。但是在这里，网络能够一直“看着图片”。

具体实现方式是：$$A_{gate} = x \cdot W_x + prev\_h \cdot W_h + \mathbf{attn \cdot W_{attn}} + b$$

没错，就是只是多了一项。而且这一项其实和最开始多层感知机中的L1、L2距离是一样的，就是能一定程度上进行反应，但是背后深究的话其实不合适，相当于是有了这个想法，但是算法没有能够匹配上。讲解一下这个式子的话就是单纯的将上一次的输出prev_h，去和提取过的网络进行一个点积，点积完了之后的矩阵，哪个部分大，哪个部分小，其实就是一个很好的“视觉焦点”的体现，那么我们将这个“视觉焦点”也进行一个加权求和变换，放进最终的求和中。



#### Teacher Forcing

这可能是一个RNN的special的训练方式，因为刚刚说了是串行的网络，为了提高训练速度，在训练的时候，是“拿着答案“去看的，



| **类名**            | **简单介绍**                                                 |
| ------------------- | ------------------------------------------------------------ |
| **`ImageEncoder`**  | **图像特征提取器**。使用 torchvision 提供的预训练 RegNet-X 模型作为主干，提取图像的 4x4 空间网格特征（c5 特征），供后续模型使用。 |
| **`WordEmbedding`** | **词嵌入层**。实现了简化版的 `torch.nn.Embedding`，负责将输入的离散词汇索引（Index）映射为连续的稠密词向量。 |
| **`RNN`**           | **单层普通循环神经网络（Vanilla RNN）**。封装了底层的单步和全序列前向传播逻辑，维护并更新隐藏状态。 |
| **`LSTM`**          | **单层长短期记忆网络**。纯底层实现，包含了输入门、遗忘门、输出门和细胞状态（Cell State）的详细计算逻辑，用于缓解 RNN 的长依赖问题。 |
| **`AttentionLSTM`** | **带有注意力机制的 LSTM**。在常规 LSTM 结构的基础上，每次更新状态前都会先计算当前隐藏状态与图像各个局部区域的匹配度（注意力），从而在生成词汇时“聚焦”到图像的特定部分。 |
| **`CaptioningRNN`** | **图像描述总控模型**。将上述的 `ImageEncoder`、`WordEmbedding` 和任意一种序列模型（RNN/LSTM/AttentionLSTM）整合在一起。提供计算训练损失的 `forward` 方法，以及用于测试阶段逐词生成句子的 `sample` 自回归生成方法。 |

| **函数名**                      | **简单介绍**                                                 |
| ------------------------------- | ------------------------------------------------------------ |
| **`hello_rnn_lstm_captioning`** | 简单的测试函数，用于确认该 Python 文件是否被成功导入。       |
| **`rnn_step_forward`**          | **RNN 单步前向传播**。计算并返回单个时间步的下一个隐藏状态（基于 `tanh` 激活函数），同时保存反向传播所需的 `cache`。 |
| **`rnn_step_backward`**         | **RNN 单步反向传播**。手动计算 Vanilla RNN 单个时间步的梯度，包括输入、前一隐藏状态以及权重矩阵的梯度。 |
| **`rnn_forward`**               | **RNN 序列前向传播**。通过一个 `for` 循环，在时间维度上不断调用 `rnn_step_forward`，处理整段时序数据。 |
| **`rnn_backward`**              | **RNN 序列反向传播**。时间倒流，通过逆序循环调用 `rnn_step_backward`，累加所有时间步对权重矩阵的梯度，并计算向后传递的误差。 |
| **`temporal_softmax_loss`**     | **时间序列损失函数**。针对时序数据的交叉熵损失函数。它计算预测词汇和真实词汇之间的差异，并在批次（Batch）和时间步（Timestep）上进行汇总，同时可以忽略用于填充的 `<NULL>` 标记。 |
| **`dot_product_attention`**     | **缩放点积注意力计算**。计算当前的隐藏状态（Query）与图像各网格特征（Key/Value）之间的注意力得分（Softmax 归一化），并返回加权融合后的视觉上下文向量以及注意力权重热力图。 |
