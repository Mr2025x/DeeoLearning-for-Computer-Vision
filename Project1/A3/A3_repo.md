## Fully_connected_networks.py

这一块其实还是线性层组成的神经网络，不过基本上是把这一块的线性网络层做到头了（起码是在作业里到头了）还是一样先列一下所有涉及的类其实就清楚很多：

| **类名 **               | **类别 **  |
| ----------------------- | ---------- |
| **`Linear`**            | 基础网络层 |
| **`ReLU`**              | 激活函数层 |
| **`Linear_ReLU`**       | 复合结构层 |
| **`Dropout`**           | 正则化层   |
| **`TwoLayerNet`**       | 网络架构   |
| **`FullyConnectedNet`** | 网络架构   |

像是Linear和ReLU层就是前面反复有在进行过构造的两个核心实现。然后这里其实主要就干了两件事：集成封装/优化

#### 集成封装

这一块和代码的名字也是符合的，就是fully。通过定义好类，进行”Linear_ReLU“的双层封装，然后在FullyConnectedNet中就能完成很快的调用，最后通过传入参数就能很快的实现想搭几层就搭几层的效果。

在具体代码的实现上，这里也没有很新的点，其实更多的是规范化的书写，比如对于最核心的”\_init_“，”forward“，”backward“；再比如在最终的神经网络层中，最后一层是Linear层这样。这一个搭建过程一直到后面也会一直延续，其实到了这里也有一点点自己的小心得：在我看来其实最复杂的是backward，但是最不重要的也是backward。主要的实现方法，或者说各种各样的创新的点，都是在forward，\_init\_上，backward更像是一种比较固定的实现（因为毕竟就是反向传播，用梯度更新参数，遵循链式法则），而且后期似乎还有自动微分器，能跳过手推公式环节（虽然还没有实现过）。而forward的实现过程是很直观的，不仅是网络的搭建还是后面的优化。所以这样来看这些网络就会比较轻松。

#### 优化

先说总的：其实这一块优化的过程的idea来源还是比较好想到的（可以这么说吗），因为上述网络中，让人最难受的就是”Learning_rate“和"weight"这两个超参数，这里之前居然是用人的经验来进行设置的，这个肯定是要被优化掉的。

于是这里进行的就是对于Learning_rate的优化，而后面那篇里的Kaiming_init就是对于weight的优化。下面是具体实现：

| **优化器**       | **核心思想与特点**                                           | **状态变量说明**                                             | **核心数学更新公式**                                         |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Vanilla SGD ** | 直接沿着当前计算梯度的反方向迈出固定的一步。容易在峡谷地形中震荡或陷入局部最优。 | 无额外状态变量                                               | $w_{t+1} = w_t - \eta \cdot dw_t$                            |
| **SGD Momentum** | **引入物理“惯性”**。 保留上一时刻的更新方向，如果在同方向连续更新则加速，在震荡方向则相互抵消，从而加速收敛。 | $v$: 速度变量 (velocity) $\mu$: 动量衰减率 (momentum)        | $v_{t+1} = \mu \cdot v_t - \eta \cdot dw_t$ $w_{t+1} = w_t + v_{t+1}$ |
| **RMSProp**      | **自适应学习率**。 累积历史梯度的平方。梯度大的方向惩罚大（步子变小），梯度小的方向惩罚小（步子变大），有效缓解震荡。 | $cache$: 梯度平方的移动平均 $\alpha$: 衰减率 (decay_rate)    | $cache_{t+1} = \alpha \cdot cache_t + (1 - \alpha) \cdot dw_t^2$ $w_{t+1} = w_t - \frac{\eta}{\sqrt{cache_{t+1}} + \epsilon} \cdot dw_t$ |
| **Adam **        | **集大成者**。 同时结合了 Momentum（一阶矩，记录方向）和 RMSProp（二阶矩，记录幅度），并引入了偏差校正来解决训练初期的冷启动问题。 | $m$: 一阶矩 (类似动量) $v$: 二阶矩 (类似梯度平方) $\beta_1, \beta_2$: 衰减率 $t$: 当前迭代步数 | **1. 算移动平均:** $m_t = \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot dw_t$ $v_t = \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot dw_t^2$ **2. 偏差校正:** $\hat{m}_t = \frac{m_t}{1 - \beta_1^t}$ $\hat{v}_t = \frac{v_t}{1 - \beta_2^t}$ **3. 更新参数:** $w_{t+1} = w_t - \frac{\eta}{\sqrt{\hat{v}_t} + \epsilon} \cdot \hat{m}_t$ |

让ai总结了一下直接把很多想写的直接写了，不过我还是写一下我自己的理解：

##### SGD

就是我之前说的要自己设置相应的学习率的基础算法，而后面的就是往这上面加东西（感觉很像数学建模）

##### SGD+Monentum

引入了”惯性“一说，通过记录”历史上“的速度（其实就是求和来实现，不过这里想成向量的首尾相加，不要简单想成是正负号的相加），来实现相应的加速/减速，能有效减小震荡，并且更快收敛。不过这里的摩擦系数还是人为设定的，相关的学习率也还是不变的

##### RMSPro

这里则是另一种方法来改变步子的大小：结合之前的步子大小，太大了就适当减小，太小了就适当加速。在这里学习率真正进行了自适应

##### Adam

综合了上面两种优化算法，既有历史记忆，又能让学习率进行自适应，并解决了结合之后的启动问题，能够较为精准+快速地到达对应目的地

#### 正则化

在这里引入了drop层来实现进一步的正则化。正则化是一种用来一定程度上缓解过拟合的方法。之前的正则化方法是通过在loss函数中，增加对于权重方差的惩罚来实现的，避免权重集中在某些系数上。这里则是通过”随机失活“来实现。在训练过程中每个神经元都有一个失活概率，从而我们的模型能够更加稳定。稍稍要提一句的是这里有一个细节是要保证失活后的信号强度依旧不变，也就是有些该输出的不输出了之后，剩下的强度要增加。但是这里其实我不是很理解了，这样最终的网络在test的时候是不引入drop层的。

## Convolutional_Network.py

这里就到卷积神经网络了，算是到了一个新的篇章。核心词肯定就是卷积了，卷积的核心作用是特征提取，记住这两点，我觉得整体的理解没有很困难。整体罗列如下：

| **函数名称**                                | **功能简述**                                                 |
| ------------------------------------------- | ------------------------------------------------------------ |
| `find_overfit_parameters()`                 | 用于寻找和返回能让模型在小批量数据上快速过拟合的超参数（如 `weight_scale` 和 `learning_rate`）。 |
| `create_convolutional_solver_instance(...)` | 实例化并返回一个配置好的 `DeepConvNet` 模型及对应的 `Solver`，用于在 CIFAR-10 上进行训练。 |
| `kaiming_initializer(...)`                  | 权重初始化函数，实现了针对线性层和卷积层的 Kaiming 初始化（以及 Xavier 初始化）。 |

| **类名称**          | **包含的方法**                                               | **功能简述**                                                 |
| ------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `ThreeLayerConvNet` | `__init__`, `save`, `load`, `loss` `__init__`，` 保存 `，` 加载 `，` 丢失` | 实现了一个三层卷积神经网络。网络架构为：`conv - relu - 2x2 max pool - linear - relu - linear - softmax`。 |

| `Conv`             | `forward`, `backward` `前进 `，` 后退`                       | 卷积层的朴素实现（不依赖 `torch.nn` 模块的高级封装，仅使用基础张量运算）。 |

整体的实现思路上，还是由浅入深，先实现了相关的单层卷积网络，然后用层层堆叠实现这里的”全连接“。



这里想记录的是整体网络的架构。Conv-ReLU-PooL的单元层是实现的核心元素，先提取特征，然后压缩。在全部压缩、提取完了之后，会将最终的结果送进一个全连接层，这个时候就基本上可以不用考虑空间结构了，自然的将这里的张量展成二维的然后让线性层去提取识别。



更加具体的是实现上的话，卷积是通过卷积核在原图上的遍历实现，会增加图片通道数，而池化操作则是折叠，对于提取出来的特征进行压缩，具体方式是每一小块只保留最大值。



归一化在这里也被引入，其实不是单单为了卷积网络服务的，当层数增加之后，后续内部的输出会被消解或是极度发散偏移。归一化就是一直将整体的数据进行一个”打包“，同时这里有平移项和缩放项系数，同样会在模型中被训练到，从而能够完全实现原信息的保留。当然这里要注意归一化是对每一层出来的“数据”进行的



然后就是kaiming_init了，就是实现上面说的对于weight超参数的自动适应。具体是初始化方式原本是进行一个高斯分布的采样，然后这里根据后续不同的是卷积网络还是线性层来自动适应了。
